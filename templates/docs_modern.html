OAuth 2.0 Authentication

Authorization Code Flow

For applications that need to act on behalf of users, VoiceForge supports OAuth 2.0 with PKCE.

1. Authorization Request

GET https://api.voiceforge.ai/oauth/authorize?
  response_type=code&
  client_id=YOUR_CLIENT_ID&
  redirect_uri=YOUR_REDIRECT_URI&
  scope=transcribe:read transcribe:write&
  state=RANDOM_STRING&
  code_challenge=CODE_CHALLENGE&
  code_challenge_method=S256

2. Token Exchange

POST https://api.voiceforge.ai/oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&
code=AUTHORIZATION_CODE&
client_id=YOUR_CLIENT_ID&
code_verifier=CODE_VERIFIER&
redirect_uri=YOUR_REDIRECT_URI

Available Scopes

  • transcribe:read - View transcription history
  • transcribe:write - Create new transcriptions
  • streaming:connect - WebSocket streaming access
  • account:read - View account information

Token Usage

Authorization: Bearer YOUR_ACCESS_TOKEN

Include the access token in the Authorization header for all API requests.

YouTube Transcription

POST /api/v1/transcribe/youtube

Transcribe YouTube videos directly by providing the video URL. Perfect for content creators, educators, and researchers.

Request Body

url *

YouTube video URL to transcribe

Type: string | Format: https://youtube.com/watch?v=... or https://youtu.be/...

language

Expected language code (optional, auto-detects if not provided)

Type: string | Default: auto | Examples: en, es, fr

Code Examples

cURL 

curl -X POST http://localhost:8000/api/v1/transcribe/youtube \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://youtube.com/watch?v=dQw4w9WgXcQ",
    "language": "auto"
  }'

Python 

import httpx
import asyncio

async def transcribe_youtube(url):
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "http://localhost:8000/api/v1/transcribe/youtube",
            headers={"Authorization": "Bearer YOUR_API_KEY"},
            json={"url": url, "language": "auto"},
            timeout=300.0  # YouTube videos may take longer
        )
        return response.json()

# Usage
result = await transcribe_youtube("https://youtube.com/watch?v=dQw4w9WgXcQ")
print(f"Transcript: {result['transcription']['text']}")

WebSocket Real-time Streaming

WS ws://localhost:8000/ws/v1/transcribe

Real-time audio streaming for live transcription with speaker diarization and language detection. Perfect for live captioning, call monitoring, and interactive applications.

Connection Parameters

api_key *

Your API key for authentication

Pass as query parameter: ?api_key=YOUR_API_KEY

sample_rate

Audio sample rate in Hz

Default: 16000 | Supported: 8000, 16000, 22050, 44100

language

Expected language code

Default: auto | Examples: en, es, fr

Protocol Flow

1. Connect & Configure

Send configuration message after connecting

{
  "type": "configure",
  "config": {
    "encoding": "linear16",
    "sample_rate": 16000,
    "language": "auto",
    "interim_results": true,
    "speaker_diarization": true,
    "language_detection": true,
    "noise_reduction": true,
    "max_speakers": 5
  }
}

2. Stream Audio

Send audio chunks as base64-encoded data

{
  "type": "audio",
  "data": "base64_encoded_audio_chunk",
  "timestamp": 1672531200000
}

3. Receive Results

Get real-time transcription updates

Implementation Examples

JavaScript (Browser) 

// Connect to WebSocket
const ws = new WebSocket('ws://localhost:8000/ws/v1/transcribe?api_key=YOUR_API_KEY');

ws.onopen = () => {
  console.log('Connected to VoiceForge');
  
  // Send configuration
  ws.send(JSON.stringify({
    type: 'configure',
    config: {
      encoding: 'linear16',
      sample_rate: 16000,
      language: 'auto',
      interim_results: true,
      speaker_diarization: true
    }
  }));
};

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  if (message.type === 'transcription') {
    console.log(`${message.is_final ? '[FINAL]' : '[PARTIAL]'} ${message.text}`);
  }
};

// Stream microphone audio
const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
const mediaRecorder = new MediaRecorder(stream);

mediaRecorder.ondataavailable = (event) => {
  if (ws.readyState === WebSocket.OPEN) {
    const reader = new FileReader();
    reader.onload = () => {
      const base64 = reader.result.split(',')[1];
      ws.send(JSON.stringify({
        type: 'audio',
        data: base64,
        timestamp: Date.now()
      }));
    };
    reader.readAsDataURL(event.data);
  }
};

mediaRecorder.start(100); // Send chunks every 100ms

Python 

import asyncio
import websockets
import json
import base64
import pyaudio

async def stream_transcription():
    uri = "ws://localhost:8000/ws/v1/transcribe?api_key=YOUR_API_KEY"
    
    async with websockets.connect(uri) as websocket:
        # Send configuration
        config = {
            "type": "configure",
            "config": {
                "encoding": "linear16",
                "sample_rate": 16000,
                "language": "auto",
                "interim_results": True,
                "speaker_diarization": True
            }
        }
        await websocket.send(json.dumps(config))
        
        # Setup audio streaming
        audio = pyaudio.PyAudio()
        stream = audio.open(
            format=pyaudio.paInt16,
            channels=1,
            rate=16000,
            input=True,
            frames_per_buffer=1024
        )
        
        # Start streaming
        async def send_audio():
            while True:
                data = stream.read(1024, exception_on_overflow=False)
                base64_data = base64.b64encode(data).decode('utf-8')
                await websocket.send(json.dumps({
                    "type": "audio",
                    "data": base64_data
                }))
                await asyncio.sleep(0.1)
        
        async def receive_results():
            async for message in websocket:
                data = json.loads(message)
                if data['type'] == 'transcription':
                    status = 'FINAL' if data['is_final'] else 'PARTIAL'
                    print(f"[{status}] {data['text']}")
        
        # Run both tasks concurrently
        await asyncio.gather(send_audio(), receive_results())

asyncio.run(stream_transcription())

Troubleshooting

Common Issues

Authentication Errors (401)

API key is invalid, expired, or missing.

  • â€¢ Verify your API key is correct and active
  • â€¢ Check that you're including the Authorization header
  • â€¢ Ensure you're using the correct format: Authorization: Bearer YOUR_API_KEY
  • â€¢ Regenerate your API key if needed in the Developer Portal

File Upload Issues

Audio file cannot be processed or is rejected.

  • â€¢ Supported formats: MP3, WAV, M4A, FLAC, OGG
  • â€¢ Maximum file size: 25MB
  • â€¢ Audio should be at least 1 second long
  • â€¢ Use proper Content-Type header: multipart/form-data

WebSocket Connection Issues

Cannot connect to or maintain WebSocket streaming.

  • â€¢ Ensure you're connecting to wss://api.voiceforge.ai
  • â€¢ Include your API key in the query parameters
  • â€¢ Handle connection drops with automatic reconnection
  • â€¢ Send keep-alive pings every 30 seconds

Rate Limiting (429)

Too many requests sent in a short time period.

  • â€¢ Wait before retrying (check Retry-After header)
  • â€¢ Implement exponential backoff in your client
  • â€¢ Consider upgrading your plan for higher limits
  • â€¢ Batch multiple files instead of individual requests

Poor Transcription Quality

Transcription accuracy is lower than expected.

  • â€¢ Use higher quality audio (16kHz+ sample rate)
  • â€¢ Minimize background noise and echo
  • â€¢ Enable noise reduction feature
  • â€¢ Specify the correct language parameter
  • â€¢ Use custom vocabulary for domain-specific terms

Debug Tips

  • â€¢ Enable verbose logging in your application
  • â€¢ Test with small audio files first
  • â€¢ Use our playground to validate API behavior
  • â€¢ Check our status page for service issues
  • â€¢ Monitor response times and error rates
